FTK Imager — Présentation de l'outil

FTK Imager est un outil développé par AccessData Group (aujourd'hui sous Exterro), né au début des années 2000 comme complément léger et gratuit à la suite Forensic Toolkit (FTK). Il s'est progressivement imposé comme l'un des standards de la criminalistique numérique, utilisé par les forces de l'ordre, les tribunaux, les entreprises et les centres de formation à travers le monde.

Acquisition (Imaging)

Sa fonction principale : créer des images forensiques bit à bit de disques durs, clés USB, cartes mémoire ou partitions, dans des formats normalisés (RAW/dd, E01, AFF) — sans jamais modifier le support original.

Intégrité des preuves

Calcul de hash cryptographiques (MD5, SHA-1, SHA-256) pour vérifier l'intégrité des données copiées et prouver l'authenticité des preuves devant un tribunal.

Analyse préliminaire

Navigation dans le système de fichiers, consultation de fichiers supprimés, prévisualisation de documents/images, lecture du registre Windows et analyse de mémoire RAM — sans altérer les données.

Chaîne de conservation

Génération automatique de rapports documentant la date/heure d'acquisition, l'outil utilisé, les hashs calculés et les informations sur le support — essentiel pour la valeur légale des preuves.

En cybersécurité, FTK Imager est utilisé en investigation post-incident (intrusions, malwares, fuites de données) et s'intègre dans un écosystème d'outils complémentaires : Autopsy/Sleuth Kit, EnCase, Volatility (analyse mémoire) et Kali Linux côté offensif.

Cas Pratique — Scénario

Notre groupe est mandaté par un juge pour analyser une clé USB saisie chez un suspect impliqué dans un réseau de trafic de drogue. À l'inspection, la clé semble vide — le suspect a probablement supprimé définitivement les données qu'elle contenait. L'objectif : créer une image forensique de la clé, puis tenter de récupérer les fichiers supprimés sans jamais altérer le support original.

Étape 1 — Acquisition de la clé USB

FTK Imager est lancé sur un poste vierge de toute investigation antérieure. L'interface principale ne montre encore aucune source de données chargée.

Créer l'image forensique

Dans le menu File → Create Disk Image…, on lance l'assistant de capture.

Choisir la source

Le type de source sélectionné est Physical Drive, correspondant à la clé USB physique à analyser (\\.\PHYSICALDRIVE1).

Choisir le format

Le format E01 (EnCase Evidence File) est retenu — un standard largement répandu en criminalistique numérique pour stocker des images disque créées avec des outils comme FTK Imager.

Informations de l'évidence

Les métadonnées de l'affaire sont renseignées pour garantir la traçabilité (chain of custody) :

Case NumberForensic-USB
Evidence Number1
Unique DescriptionClé USB — Images supprimées
ExaminerGroupe4 Forensic

Destination de l'image

L'image est enregistrée dans C:\Users\user\Desktop\Forensic sous le nom usb_image_supprimées, avec une taille de fragment de 1500 Mo et un taux de compression de 6.

Lancement de la capture

Une fois la destination ajoutée à la liste (usb_image_supprimées [E01]), le bouton Start déclenche la création de l'image, avec vérification automatique après création.

Étape 2 — Chargement de l'image pour analyse

Après acquisition, la clé USB physique n'est plus utilisée. Toute l'analyse qui suit se fait exclusivement à partir de l'image forensique, afin de préserver l'intégrité de la preuve originale — principe fondamental de la chaîne de conservation.

Via Add Evidence Item, on sélectionne cette fois le type de source Image File (et non plus Physical Drive), car on travaille désormais sur une copie forensique.

La création de l'image a généré plusieurs segments (.E01, .E02, .E03, .E04) dans Bureau\Forensic. Seul le premier segment E01 doit être sélectionné — FTK Imager reconstitue automatiquement l'ensemble.

Étape 3 — Recherche des fichiers supprimés

Dans la fenêtre principale de FTK Imager, l'Evidence Tree affiche l'arborescence du volume SAV-USB [FAT32]. Plusieurs fichiers marqués d'une croix rouge (✕) apparaissent comme supprimés mais restent récupérables via leur File Slack — l'espace résiduel d'un cluster non réécrit par le système de fichiers.

Les fichiers retrouvés (bateau de livraison.jpg, chanvre indien.jpg, cocaine-Maroc.jpg, cocaine-Senegal.jpg, etc.) confirment la piste évoquée dans le scénario : ces images n'avaient pas été physiquement écrasées et demeuraient accessibles dans l'espace non alloué du disque.

Étape 4 — Export des fichiers retrouvés

Les fichiers d'intérêt sont sélectionnés dans la liste, puis un clic droit ouvre le menu contextuel pour lancer Export Files… vers un répertoire de sortie dédié.

Résultat de l'export

L'analyse de l'image forensique a permis de récupérer plusieurs fichiers définitivement supprimés de la clé USB, regroupés dans le dossier Image-Forensic Récupérée sur le Bureau. Ces preuves numériques corroborent l'implication du suspect dans le réseau visé par l'enquête.

Conclusion du cas pratique : ce TP illustre un principe fondamental de la forensique numérique — la suppression d'un fichier n'efface pas immédiatement son contenu physique sur le support. Tant que les clusters concernés ne sont pas réécrits, les données restent récupérables via l'analyse du File Slack et de l'espace non alloué, ce qui permet de reconstituer fidèlement des preuves qu'un suspect croyait avoir détruites.
