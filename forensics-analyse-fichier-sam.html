Analyse Forensique du Fichier SAM

Forensics & Investigation · Reconstitution d'activités sur une VM Windows 10 avec Autopsy · TP n°001

Autopsy 4.22.1 Windows 10 x64 (.vmdk) Fichier SAM Web History / Downloads MD5 / SHA-256

Ce TP de forensique numérique reconstitue trois activités réalisées intentionnellement sur une machine virtuelle Windows 10 : la navigation web, le téléchargement de logiciels, et surtout la suppression volontaire du fichier SAM (Security Account Manager), base de données critique des comptes Windows. L'objectif : démontrer qu'aucune action sur un système n'est jamais totalement invisible pour un analyste équipé des bons outils.

Objectifs de l'Analyse

Trois activités ont été réalisées délibérément sur la VM, puis recherchées via Autopsy, comme le ferait un analyste lors d'une investigation réelle :

Navigation web

Connexion à 3 sites différents depuis Microsoft Edge.

Téléchargements

Récupération de 3 logiciels depuis Internet.

Suppression du SAM

Effacement de la base de comptes Windows via l'Invite de commandes en mode récupération (WinRE).

Étape 1 — Suppression du fichier SAM sur la VM

Avant toute analyse Autopsy, le fichier C:\Windows\System32\config\SAM a été supprimé directement depuis la VM, via l'Invite de commandes en mode récupération Windows (WinRE) — la seule façon d'y accéder, ce fichier étant verrouillé par le système tant que Windows est démarré normalement.

Accès au mode récupération

Le chemin classique : Paramètres → Mise à jour et sécurité → Récupération → Démarrage avancé → Résolution des problèmes → Options avancées → Invite de commandes, avec authentification du compte utilisateur requise pour ouvrir un terminal Administrateur.

Suppression et confirmation

X:\windows\system32> D: dir
D:\> cd Windows\System32\config
D:\Windows\System32\config> del SAM
D:\Windows\System32\config>

La commande dir confirme ensuite l'absence du fichier SAM dans le répertoire — la suppression est effective au niveau du système de fichiers actif.

Impact immédiat sur le système

Au redémarrage, Windows ne peut plus accéder à sa base d'authentification : un écran bleu (BSOD) apparaît avec le code d'arrêt 0xc00002e3, suivi du déclenchement automatique de la réparation système.

Remarque : dans un contexte réel, la suppression du fichier SAM constitue une action malveillante grave visant à détruire les preuves d'authentification d'un système — un signal d'alerte majeur pour tout analyste forensique.

Résultats de l'Analyse

5.1 — Le fichier SAM dans Autopsy

Le fichier SAM (Security Account Manager) est un fichier système critique de Windows : il stocke les hashs des mots de passe des comptes utilisateurs locaux, à l'emplacement C:\Windows\System32\config\SAM.

Dans Autopsy, la navigation vers le dossier config de la VM confirme que le fichier porte les traces de sa modification — preuve numérique de l'action de suppression, même si le fichier reste référencé dans le système de fichiers analysé.

5.2 — Traces de navigation web (Web History)

Autopsy a extrait 65 entrées d'historique web depuis Microsoft Edge. Les 3 sites visités intentionnellement ont été retrouvés :

Conclusion

L'analyse forensique de la VM Windows 10 avec Autopsy 4.22.1 a permis de retrouver avec succès les traces des trois activités du TP : les 3 sites web visités parmi 65 entrées d'historique, les 3 logiciels téléchargés avec leurs URLs et dates exactes, et la suppression du fichier SAM documentée étape par étape — jusqu'à son impact direct (BSOD, code 0xc00002e3).

Ce TP illustre un principe fondamental de la forensique numérique : toute action sur un système laisse des traces. Même une tentative de destruction de preuves génère elle-même de nouvelles traces (logs système, BSOD, métadonnées de modification). Un analyste qualifié peut ainsi reconstituer fidèlement la chronologie des événements.
