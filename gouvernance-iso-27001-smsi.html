Mise en place d'un SMSI – ISO 27001

📋 ISO 27001 🔐 SMSI 📊 Analyse de risques 📄 Déclaration d'Application Exercice individuel

🏢 Contexte – DIGISEN SARL

En tant que consultant sécurité pour DIGISEN SARL (services numériques : hébergement + développement web, 50 employés), l'objectif est de préparer cette entreprise à la certification ISO 27001 en mettant en place les bases d'un SMSI (Système de Management de la Sécurité de l'Information).

Données sensibles en jeu : données clients, applications web hébergées, données RH.

2Identification des menaces et vulnérabilités

ActifMenacesVulnérabilités
Base de données clients Fuite de données, Injection SQL Absence de chiffrement, patching insuffisant
Applications Web hébergées DDoS, exploitation de failles applicatives Code non audité, absence de WAF
Données RH Vol d'identité, accès non autorisé Droits mal configurés, absence de journalisation

3Matrice de risques (C = Impact × Probabilité)

ActifMenaceImpact (1-5)Probabilité (1-5)Criticité
Base de données clients Fuite de données 5 4 20 🔴
Données RH Accès non autorisé 4 4 16 🟠
Applications Web DDoS 4 3 12 🟡

4Traitement des risques

🔴 Fuite de données (20) → RÉDUIRE
Chiffrement de la base de données, durcissement du SGBD, tests d'intrusion réguliers, patch management strict
🟠 Accès non autorisé RH (16) → ÉVITER
Contrôle d'accès strict basé sur les rôles (RBAC), MFA obligatoire, journalisation et alertes sur les accès suspects
🟡 DDoS sur applications web (12) → TRANSFÉRER
Contrat avec fournisseur anti-DDoS (Cloudflare), WAF en amont des applications, SLA avec le prestataire

6Déclaration d'Application (SoA)

La SoA (Statement of Applicability) liste les contrôles ISO 27002 applicables à DIGISEN et leur justification.

Domaine ISO 27002ContrôleApplicabilitéJustification
A.9 – Gestion des accès MFA, droits minimaux ✅ Applicable Réduit le risque d'accès non autorisé aux données RH
A.12 – Sécurité opérationnelle Patch management, sauvegardes ✅ Applicable Réduit le risque de fuite de données clients
A.13 – Sécurité des communications Chiffrement TLS, VPN ✅ Applicable Protège les échanges sensibles avec les clients
A.14 – Acquisition & développement Tests de sécurité applicative ✅ Applicable Réduit les vulnérabilités des applications web hébergées
A.17 – Continuité Anti-DDoS, PRA/PCA ✅ Applicable Transfert/réduction du risque DDoS
A.18 – Conformité Journalisation et traçabilité ✅ Applicable Obligations légales et audit interne
A.6 – Organisation de la sécurité Politique de sécurité ✅ Applicable Nécessaire pour formaliser le SMSI
A.11 – Sécurité physique Contrôle d'accès aux locaux ✅ Applicable Protection des serveurs hébergés

✅ Ce que j'ai appris

  • Mener une analyse de risques structurée : identifier les actifs, les menaces, les vulnérabilités, puis calculer la criticité (I × P)
  • Distinguer les 4 stratégies de traitement des risques : Réduire, Éviter, Transférer, Accepter
  • Cartographier les contrôles ISO 27002 selon les risques identifiés et rédiger une Déclaration d'Application (SoA)
  • Comprendre que l'ISO 27001 est une norme certifiable — la SoA est un livrable obligatoire pour l'audit
  • Voir la complémentarité avec le NIST : l'ISO certifie, le NIST guide — les deux ensemble forment un SMSI robuste
